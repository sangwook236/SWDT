[-] Usage.
	- Get started.
		https://docs.openclaw.ai/start/getting-started

		Run onboarding:
			openclaw onboard --install-daemon
		Check the Gateway:
			openclaw gateway status

			openclaw gateway start
			openclaw gateway restart
				openclaw gateway stop
		Open the Control UI:
			openclaw dashboard

		https://docs.openclaw.ai/start/wizard

		CLI onboarding is the recommended way to set up OpenClaw on macOS, Linux, or Windows (via WSL2; strongly recommended).
		It configures a local Gateway or a remote Gateway connection, plus channels, skills, and workspace defaults in one guided flow.
			openclaw onboard
		Reconfigure:
			openclaw configure
				openclaw configure --section web
			openclaw agents add <name>

	- Providers.
		https://docs.openclaw.ai/providers

		Ollama:
			https://docs.openclaw.ai/providers/ollama

			Pull a local model:
				ollama pull gpt-oss:20b

			Sign in if you want cloud models:
				ollama signin

			Run onboarding:
				openclaw onboard
				openclaw onboard --auth-choice ollama

				Non-interactive mode:
					openclaw onboard --non-interactive --auth-choice ollama --accept-risk
				Optionally specify a custom base URL or model:
					openclaw onboard --non-interactive --auth-choice ollama --custom-base-url "http://ollama-host:11434" --custom-model-id "qwen3.5:27b" --accept-risk

			If you prefer manual setup, enable Ollama for OpenClaw directly:
				Any value works; Ollama doesn't require a real key.

				Set environment variable:
					export OLLAMA_API_KEY="ollama-local"
					export OLLAMA_HOST="0.0.0.0"

				Or configure in your config file:
					openclaw config set models.providers.ollama.apiKey "ollama-local"

			Set an Ollama model as the default model:
				openclaw config set agents.defaults.model.primary "ollama/llama3.3"

			Inspect or switch models:
				ollama list
				openclaw models list
				openclaw models set ollama/glm-4.7-flash

				openclaw gateway status

				Or set the default in config:
					{
					  agents: {
						defaults: {
						  model: { primary: "ollama/glm-4.7-flash" },
						},
					  },
					}

				Kill anything bound to the default gateway port, then start it.
			openclaw gateway ...
				Gateway control via WebSocket.

[-] Troubleshooting.
	- <error> gateway.mode is unset; gateway start will be blocked.
			Fix: run openclaw configure and set Gateway mode (local/remote). 
			Or set directly: openclaw config set gateway.mode local
			Missing config: run openclaw setup first.
		<solution> openclaw config set gateway.mode local
